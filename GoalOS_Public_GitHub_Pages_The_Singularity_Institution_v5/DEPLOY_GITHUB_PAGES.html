Fastest deployment

  1. Extract the deployment ZIP.
  2. Create or clear the intended GitHub repository.
  3. Upload the extracted contents to the repository root—not a containing folder.
  4. In Settings → Pages, publish from the default branch and repository root, or enable the included Actions workflow.
  5. Wait for deployment, then enforce HTTPS.
  6. Complete the live-site checklist below.

Live-site checklist

  • Hero, cover, typography, and document cards render with styling.
  • All 23 document links open.
  • Search and filters operate.
  • Proof Loop Studio reaches a terminal decision state.
  • Governed RSI controls update the score and disposition.
  • Terms modal appears before protected downloads and demonstrations.
  • English and French legal pages open.
  • No console errors, mixed-content warnings, or broken images.
  • Mobile layout has no root horizontal overflow.
  • Deployment commit and SHA-256 release checksum are recorded.

Important publication boundary

GitHub Pages is public. Do not add private keys, credentials, customer data, confidential inventions, privileged advice, signer or recovery details, internal valuation models, token-sale materials, or operator-specific incident information to the repository.
