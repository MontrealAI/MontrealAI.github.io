Operating architecture

The system in four functions.

P

Private

Underlying intelligence remains encrypted, off-chain or inside protected execution.

E

Evidence

Inclusion proofs and attestations make authority inspectable.

R

Root

Changing one object changes the committed root.

F

Future

Only admitted, scope-compatible skills under the current root influence missions.

Three layers

Private capability: methods, traces and sensitive context. Proof artifacts: Dockets, ProofBundles, attestations and decisions. Public commitment: roots, hashes, policies, identity and history.

Typed Merkle forest

Distinct roots for skills, edges, evidence, ProofBundles, policies, attestations, revocations, supersessions and metadata are bound into one epoch root.

The root is not truth

A root proves that a canonical state was committed. Chronicle, Dockets, validators and policies decide whether that state deserves authority.

Private reuse

An entitled agent can invoke capability and receive a bounded result without receiving the method, capsule, keys or private graph.
