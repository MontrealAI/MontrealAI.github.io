# GoalOS Partner MASTERCLASS Architecture

## 1. Orchestration layer

`START_HERE.html` is the command surface. It records the intended audience, objective, risk, evidence maturity, desired outcome and delivery duration, then generates a live run of show.

## 2. Presentation plane

- **APEX Boardroom:** category and executive decision
- **Masterpiece Executive Deck:** canonical 32-slide synthesis
- **Grand Deck:** complete 72-slide institution
- **Sovereign Deck:** evidence, RSI and transfer
- **Production Showcase Deck:** concise commercial narrative

## 3. Operating plane

- Mission Contract
- Proof Debt
- mission-specific AGI Jobs
- ProofBundles
- Evidence Docket
- independent validation / AGI Node Council
- Governed Decision State
- Action Graph
- Chronicle
- Validated Skill Passport

## 4. Evidence plane

### First-party real

GOALOS-REAL-001 contains a public task snapshot, source mission, scoped skill, equal-constraint comparison, Docket, graph root and first-party replay receipt. Independence remains a separate gate.

### Deterministic reference

COMP-001 and Proof Run fixtures test schemas, gates, Chronicle admission, Merkle state and local reproducibility.

### Illustrative

The strategic-partnership mission demonstrates the product and decision package without representing the sample metrics as external outcomes.

## 5. Governed improvement plane

`TARGET → EMIT → FILTER → ATLAS → TEST-PLAN → EVAL → INSERT → PROMOTE`

Hard requirements include state continuity, drift detection, Evidence Contact Index, baseline comparison, replay, stress, persistence, validator independence, scope, canary and rollback.

## 6. Capability-memory plane

Chronicle is the memory firewall. An admitted capability carries stable identity, version, scope, allowed and excluded uses, method, evidence, tests, validators, replay, risk, freshness, rollback, revocation, supersession and mission lineage.

Merkle commitments preserve exact admitted state; they do not independently grant truth or unrestricted authority.

## 7. Economic-accountability plane

`$AGIALPHA` is presented as the bonded authority rail around accepted work: execution, validation, challenge, reuse, node eligibility and governed upgrades. The customer-facing path may remain fiat-denominated while professional operators supply protocol security.

## 8. Partner-conversion plane

`Proof Mission Sprint → Enterprise Proof OS → Private Validated Skill Graph → Managed AGI Node Council → Cross-Institution Proof Network`

The first engagement is bounded, reversible and reviewable. Expansion is earned through repeated evidence and measurable future-mission contribution.

## 9. Participation plane

The versioned v2.1 receipt binds organization, representative, authority, professional capacity, French-first delivery, language election, terms hashes and calibrated acknowledgements. Protected commercial routes remain gated until a matching receipt exists.
