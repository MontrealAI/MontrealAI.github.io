# SkillOS Meeting Run of Show

## Objective

Leave the meeting with alignment, named owners, and a clear path from public proof to Pilot 001.

## 45-minute flow

1. Open the public archive.
2. Show the live proof page.
3. Show the GitHub repository.
4. Explain the loop: Work -> Trace -> Learn -> Skill -> Test -> Release -> Improve.
5. Explain Pilot 001.
6. Ask for 1-3 pilot partners or warm introductions.
7. Assign owners and dates.

## Close

SkillOS proves the mechanism. Pilot 001 proves the market.
