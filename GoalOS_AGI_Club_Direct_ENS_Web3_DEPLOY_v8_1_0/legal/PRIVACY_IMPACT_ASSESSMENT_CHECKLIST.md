# Privacy Impact Assessment checklist

This checklist supports—not replaces—the privacy impact assessment required for the actual deployment and operating context.

- [ ] Name the accountable privacy officer and publish contact details.
- [ ] Map every personal-information element, purpose, source, recipient, location, retention period, and deletion method.
- [ ] Confirm that the static build contains no analytics, tracking pixels, or automatic form transmission.
- [ ] Confirm that wallet addresses and ENS names are used only for stated membership purposes.
- [ ] Minimize collection and avoid sensitive personal information in public/static tools.
- [ ] Assess transfers or access outside Québec/Canada and execute required written agreements.
- [ ] Document consent language separately and in plain terms.
- [ ] Define access, correction, withdrawal, deletion, complaint, and portability handling.
- [ ] Define incident assessment, notification, and incident-register procedures.
- [ ] Re-run the assessment after material architecture, provider, data, or purpose changes.

Qualified Québec/Canadian privacy counsel should review the completed assessment before launch.
