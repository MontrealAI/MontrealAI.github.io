Sovereign Agentic AGI ALPHA - Deployment Instructions

Welcome! This page explains how to deploy the Sovereign Agentic AGI ALPHA Agent in a user-friendly manner. You only need two files:

  1. This HTML (sovereign_agentic_agialpha_agent_deployment_instructions.html) with instructions
  2. A shell script (deploy_sovereign_agentic_agialpha_agent.sh) that creates all the needed project files/folders for you

Prerequisites: Ensure Docker & Docker Compose are installed. If you plan to run without Docker, make sure Python 3.9+ is available.

Quick Start

  1. Download / Save the script: Download Deployment Script (Right-click “Save Link As” if needed.)
  2. Make it executable: 
    chmod +x deploy_sovereign_agentic_agialpha_agent.sh
  3. Run it: 
    ./deploy_sovereign_agentic_agialpha_agent.sh

    It will automatically create a sovereign_agent_deploy folder (or any folder you choose), generate all required files (the monolithic sovereign_agent.py, Docker config, requirements.txt, trust_whitelist.yaml, etc.), and then run docker-compose build + docker-compose up -d.

  4. Attach / Inspect logs: 
    docker-compose logs -f agent
    Or to open an interactive console: 
    docker attach sovereign_agent
  5. Stop everything: 
    docker-compose down
    This command tears down the containers.

Editing Trust & Config

If you want to modify domains, paths, or tools that the agent can use, open the newly created trust_whitelist.yaml inside sovereign_agent_deploy. This sets the startup trust config for MCP discovery, local file writes, and more.

Memory Backend / LLMs

By default, the script sets environment variables so that Chroma is used for memory. If you want Weaviate, you can either edit the docker-compose.yaml in sovereign_agent_deploy or set MEMORY_BACKEND=weaviate before running docker-compose up. For LLM usage (OpenAI, Anthropic, or Ollama), set LLM_PROVIDER and relevant API keys (OPENAI_API_KEY, ANTHROPIC_API_KEY, or OLLAMA_MODEL).

Local Python (No Docker) Option

If Docker isn't available, you can run: 

pip install -r requirements.txt
python sovereign_agent.py

But you still need the generated files from the script (or copy them manually).

Security Reminders

  • Always carefully set trust_whitelist.yaml. The agent is extremely powerful!
  • Keep Docker images updated for security patches.
  • For large local LLMs, ensure your system can handle CPU/GPU usage as required.

That’s it! Once you’ve run deploy_sovereign_agentic_agialpha_agent.sh, the agent is up and running in containers, ready to deliver the absolute theoretical ALPHA SIGNAL from all MCPs and surpass normal cognitive limits. Good luck!
