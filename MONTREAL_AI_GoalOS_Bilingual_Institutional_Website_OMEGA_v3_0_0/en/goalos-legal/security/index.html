SECURITY AND RESPONSIBLE DISCLOSURE

Hardened static surface; no certification, bounty or universal safe harbour is promised.

Public surface

The site is static with no database, account, session, wallet, API, server form, analytics or client secret. A restrictive CSP limits scripts, connections, objects, forms and frames.

Confidential report

Report a potential vulnerability to secretariat@montreal.ai with subject “Security / Sécurité.” Include version, URL, component, minimal steps, impact and possible mitigation. Do not include unnecessary secrets or third-party data.

Expected conduct

Do not exceed what is necessary, access another person’s data, disrupt service, move assets, compromise systems or publish before a reasonable remediation period.

No implied bounty or safe harbour

This coordination request creates no bounty, immunity, employment, agency or duty to pay, investigate or remediate. A separate programme must be expressly published.

Deployments

Any party deploying a GoalOS system is responsible for threat modelling, code review, dependencies, SBOM, secrets, keys, access, logging, backup, recovery, testing, audit, privacy, compliance and incident response.
